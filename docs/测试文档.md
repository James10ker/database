# 校园活动管理系统测试文档

## 1. 测试目标

本测试文档用于验证校园活动管理系统是否实现需求文档中的核心功能，包括用户登录、活动管理、报名管理、签到管理、评分反馈、统计查询、活动推荐、管理员管理以及数据库 SQL 对象演示。

## 5. 数据库 SQL 测试

### 5.1 视图测试

测试 SQL：

```sql
SELECT *
FROM vw_activity_summary
WHERE activity_id = 5;
```

预期结果：

- 返回活动统计数据。
- 包含报名人数、签到人数、签到率、平均评分。

测试 SQL：

```sql
SELECT *
FROM vw_student_participation
WHERE student_no = '2026001';
```

预期结果：

- 返回学生参与活动记录。
- 包含报名状态、签到状态、评分等信息。

### 5.2 存储过程测试

测试 SQL：

```sql
EXEC usp_activity_statistics 5;
```

预期结果：

- 返回活动 5 的统计数据。
- 示例结果：报名人数 2，签到人数 2，签到率 100.00，平均评分 4.50。

测试 SQL：

```sql
EXEC usp_register_activity
    @student_id = 4,
    @activity_id = 1;
```

预期结果：

- 如果活动可报名且未重复报名，返回新报名编号。
- 如果不满足条件，抛出对应错误。

### 5.3 触发器测试

测试步骤：

```sql
UPDATE activity
SET capacity = 1
WHERE activity_id = 1;

INSERT INTO registration (activity_id, student_id, registration_status)
VALUES (1, 5, 'registered');
```

预期结果：

```text
Registration count cannot exceed activity capacity.
```

测试后恢复：

```sql
UPDATE activity
SET capacity = 80
WHERE activity_id = 1;
```

## 6. 接口测试

### 6.1 登录接口

```powershell
$body = @{
  username = '2026001'
  password = '123456'
  role = 'student'
} | ConvertTo-Json

Invoke-RestMethod -Uri 'http://localhost:8080/api/login' `
  -Method Post `
  -ContentType 'application/json' `
  -Body $body
```

预期结果：

- `code = 200`
- 返回 `token`
- 返回用户角色 `student`

### 6.2 活动列表接口

```powershell
Invoke-RestMethod -Uri 'http://localhost:8080/api/activities' -Method Get
```

预期结果：

- 返回活动列表。
- 包含 `organizerName`、`statusText`、`registrationCount`、`remainingCapacity` 等字段。

### 6.3 活动统计接口

```powershell
Invoke-RestMethod -Uri 'http://localhost:8080/api/statistics/activity/5' -Method Get
```

预期结果：

- 返回报名人数、签到人数、签到率、评分人数、平均评分。

## 7. 测试结论

根据以上测试，系统能够完成文档中设计的主体功能：

- 学生、组织者、管理员三类角色登录。
- 活动发布、浏览、详情、报名、取消报名。
- 报名名单查看。
- 签到码签到和人工补签。
- 活动评分反馈。
- 活动统计和推荐。
- 数据库表、约束、索引、视图、存储过程、触发器均可执行和演示。

系统满足课程设计汇报和演示要求。
