# 校园活动管理系统部署指南

## 1. 系统说明

本系统为前后端分离的校园活动管理系统，包含学生端、组织者端和管理员端。系统使用 Vue 3 + Vite + Element Plus 构建前端，使用 Spring Boot 3 + MyBatis-Plus 构建后端，数据库使用 SQL Server Express。

## 2. 运行环境

| 环境 | 要求 |
|---|---|
| 操作系统 | Windows |
| 数据库 | SQL Server Express，实例名建议为 `.\SQLEXPRESS` |
| 数据库工具 | SSMS 或 `sqlcmd.exe` |
| JDK | Java 17 及以上，当前项目已验证 Java 21 可用 |
| Maven | Maven 3/4 均可 |
| Node.js | Node.js 18 及以上，当前项目已验证 Node 22 可用 |
| npm | npm 9 及以上 |

## 3. 项目目录

```text
database/
├─ backend/              Spring Boot 后端
├─ frontend/             Vue 前端
├─ sql/init.sql          数据库初始化脚本
├─ docs/api.md           接口文档
├─ docs/部署指南.md      部署说明
├─ docs/测试文档.md      测试说明
└─ start-all.ps1         一键启动脚本
```

## 4. 数据库部署

### 4.1 使用脚本初始化

在项目根目录执行：

```powershell
sqlcmd -S .\SQLEXPRESS -E -i .\sql\init.sql
```

该脚本会创建数据库 `campus_activity`，并完成以下内容：

- 创建学生、组织者、管理员、活动、报名、签到、评分 7 张核心表。
- 创建主键、外键、唯一约束和 CHECK 约束。
- 创建查询索引。
- 插入测试数据。
- 创建视图、存储过程和触发器。

### 4.2 数据库对象

| 类型 | 名称 |
|---|---|
| 数据库 | `campus_activity` |
| 表 | `student`, `organizer`, `admin_user`, `activity`, `registration`, `checkin`, `rating` |
| 视图 | `vw_activity_summary`, `vw_student_participation` |
| 存储过程 | `usp_activity_statistics`, `usp_register_activity` |
| 触发器 | `tr_registration_capacity_guard` |

## 5. 后端部署

进入后端目录：

```powershell
cd backend
.\start.ps1
```

后端默认启动端口：

```text
http://localhost:8080/
```

说明：

- `backend/start.ps1` 会自动读取 SQL Server Express 的动态 TCP 端口。
- 后端使用 Windows 集成认证连接 SQL Server。
- SQL Server Windows 认证 DLL 位于 `backend/auth/`。

如需手动覆盖数据库连接，可设置环境变量：

```powershell
$env:DB_URL='jdbc:sqlserver://localhost:57961;databaseName=campus_activity;encrypt=true;trustServerCertificate=true;integratedSecurity=true'
$env:DB_USERNAME=''
$env:DB_PASSWORD=''
```

## 6. 前端部署

进入前端目录：

```powershell
cd frontend
npm install
npm run dev
```

前端默认启动端口：

```text
http://localhost:5173/
```

前端接口默认访问：

```text
http://localhost:8080/api
```

## 7. 一键启动

在项目根目录执行：

```powershell
.\start-all.ps1
```

脚本会自动完成：

- 检查数据库是否存在。
- 数据库不存在时自动执行 `sql/init.sql`。
- 检查前端依赖是否安装。
- 启动后端服务。
- 启动前端服务。
- 输出访问地址和测试账号。

如果需要重置数据库测试数据，执行：

```powershell
.\start-all.ps1 -ResetDatabase
```

如果只想启动前后端，不检查数据库：

```powershell
.\start-all.ps1 -SkipDatabase
```

## 9. 常见问题

### 9.1 前端打不开

检查端口 5173 是否启动：

```powershell
netstat -ano | findstr 5173
```

也可以重新执行：

```powershell
cd frontend
npm run dev
```

### 9.2 后端连接数据库失败

检查 SQL Server 服务：

```powershell
Get-Service -Name 'MSSQL$SQLEXPRESS'
```

检查数据库是否存在：

```powershell
sqlcmd -S .\SQLEXPRESS -E -Q "SELECT DB_ID('campus_activity');"
```

### 9.3 数据为空或测试数据不对

重新初始化数据库：

```powershell
.\start-all.ps1 -ResetDatabase
```

注意：该操作会重置数据库测试数据。
